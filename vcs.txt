jenis vcs (Version Control System)
- local: mecatat perubahan pada file di harddisk komputer sendiri, offline
- centralized: mencatat perubahan pada file yang disimpan di dalam sebuah server yang terhubung ke sebuah jaringan (LAN, NAS)
	contoh: GDrive. (penggunanya harus terhubung ke internet)
- distributed: Sama seperti centralized, tetapi seluruh file dan versinya disimpan ke komputer-komputer yg ada di jaringan tsb.
	cth: Git (bisa dianggap local uga sih), MErcurial
	
	Git: Awalnya diciptakan oleh developer-developer OS Linux.
	
	--The Three Tree--
	Modified: file yang udah diganti
	Stagged	: ditandai untuk di commit
	Commited: ditandai udah di commit
	
	Kenapa harus ada tiga status ini???: 
		- biar gak 'tabrakan'
		- Biar yakin apakah perubahan yang dilakukan itu beneran versi terbaru
		- Commit itu saat perubahannya udah FINAL!
		
	--The Three Section--
	Working directory --> git add --> Stagging area --> git commit --> Repository
	
	---Snapshot---
	- perubahan yang dibuat pada file tertentu. (progressnya)
	- Snapshot A <- snapshot B <- snapshot C
	  -> dari C bisa rollback ke B, dari B bisa rollback ke A
	  -> TAPI... kalo dari C rollback ke A ... ->snapshot B hilang.
